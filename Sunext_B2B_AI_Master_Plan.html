Sunext AI B2B - Functional MVP PRD

Bản Đặc Tả Yêu Cầu Sản Phẩm (PRD) kết hợp Functional Spec & Kiến trúc AEO/SEO (AI Engine Optimization). Mục tiêu xây dựng Sunext thành một Entity Graph chuyên gia thực thụ để AI Search có thể parse và trích dẫn.

1. Định Vị Dự Án & Phạm Vi MVP (Scope)

Bối cảnh: Sunext Pivot từ EdTech bán lẻ sang B2B AI Transformation Consulting. Website đóng vai trò kép: (1) Cỗ máy Lead Generation (CRO), và (2) Knowledge Base có cấu trúc (AEO/SEO) để các cỗ máy AI (ChatGPT Search, Perplexity, Google AI Overviews / AI Mode) nhận diện Sunext là một Entity uy tín.

Tính năng MVP (Phase 1 - 6 Tuần) Phase 2 (Sau MVP)

1.5. Non-Goals / LMS Boundary (Bắt buộc tuân thủ)

Website Sunext KHÔNG PHẢI là hệ thống LMS (Learning Management System) và không thay thế hệ thống LMS hiện có. Website này là B2B Corporate Website kết hợp hệ thống Lead Gen & AEO Knowledge Base.

Out of scope (Tuyệt đối không xây dựng trong dự án này):

  • Không có tài khoản người học / Đăng nhập để học.
  • Không có dashboard học tập (My Courses).
  • Không có trình phát bài học / video khóa học (Video Player).
  • Không có tracking tiến độ học.
  • Không có bài kiểm tra / chấm điểm học viên.
  • Không có thanh toán online khóa học (Checkout / Payment Gateway).
  • Không có cấp chứng chỉ tự động.
  • Không quản trị lớp học / giảng viên / học viên.

Website chỉ phục vụ:

  • Giới thiệu công ty Sunext (Năng lực, đội ngũ, phương pháp luận).
  • Giới thiệu các chương trình/dịch vụ AI.
  • Giới thiệu khách hàng / đối tác / case studies.
  • Thu lead B2B qua assessment, form tư vấn, gated content.
Existing LMS Integration Policy (Nút Đăng nhập):
  • Header/Footer có nút "Đăng nhập LMS" trỏ link trực tiếp sang hệ thống LMS hiện tại.
  • Behavior: Link mở tab mới (target="_blank") kèm icon external link để user biết họ đang rời khỏi trang.
  • Tracking: Bắn event GA4 lms_login_clicked trước khi chuyển trang.
  • Website Sunext KHÔNG xử lý authentication LMS. Không truyền token/session từ website sang LMS trong MVP. Không dùng chung database user.
  • KHÔNG đồng bộ tiến độ học tập trong MVP này.

3. Khung Sản Phẩm 6 Tầng (Chi tiết nội dung)

Quy ước thuật ngữ (Program vs Service)

  • Program (Chương trình đào tạo/huấn luyện): Áp dụng chủ yếu cho Tầng 1 đến Tầng 4 (Ví dụ: AI Foundation, AI Agent Bootcamp).
  • Service (Dịch vụ tư vấn/triển khai): Áp dụng chủ yếu cho Tầng 5 và Tầng 6 (Ví dụ: Tư vấn chiến lược AI-First, Triển khai Private LLM/RAG).

Đây là lõi khung nội dung để thiết kế nội dung Website. Kiến trúc website sẽ bóc tách mỗi tầng thành một URL (Entity) riêng biệt để tối ưu crawl.

Tầng 1: Low-risk

1. AI Cơ Bản (AI Foundation)

Thời lượng: 1-2 ngày. Đối tượng: Toàn bộ nhân viên.

Xóa mù AI, chuẩn hóa năng lực Prompting cơ bản (4 bước) để sửa lỗi chính tả, tóm tắt PDF, viết email.

Tầng 2: Specialization

2. AI Nâng Cao (Theo phòng ban)

Thời lượng: 1-2 ngày. Đối tượng: Chuyên viên theo cụm (HR, MKT).

Dùng công cụ trả phí (Midjourney, Advanced Data Analysis) giải quyết task chuyên sâu.

Tầng 3: Automation

3. Chatbot & Agentics

Thời lượng: 3-5 ngày. Đối tượng: Tech-savvy.

Tự tạo AI Agent nội bộ thay thế nhân viên data entry.

Tầng 4: Workflow

4. AI Vibe Coding & Automation

Thời lượng: 2-3 ngày. Đối tượng: AI Champions.

Dùng Make.com/Zapier nối các Agent lại thành luồng tự động xuyên phòng ban.

Tầng 5: Enterprise

5. LLM & Private Data

Thời lượng: Dự án tư vấn. Đối tượng: CTO, IT Director.

Build Vector DB, RAG bảo mật 100%, tích hợp ERP/CRM.

Tầng 6: The Vision

6. AI-First Enterprise

Thời lượng: Đồng hành dài hạn. Đối tượng: CEO, BOD.

Biến AI thành lợi thế cạnh tranh cốt lõi, thay đổi business model.

4. Phễu Chuyển Đổi & Kế Hoạch Lead Scoring

Mọi hành vi (Events) đều được lưu trữ trực tiếp vào DB thay vì chạy bằng "niềm tin".

  • UTM Tracking: Capture utm_source, utm_medium, utm_campaign từ URL lưu vào localStorage.
  • Ghi nhận Event: Các hành động trên web lưu trực tiếp vào bảng lead_events trong Supabase.
  • Scoring Logic (MVP):
    • Submit Assessment: +20 điểm
    • Tải Gated Content (PDF): +15 điểm / file
    • Submit "Strategy Call" Form: +50 điểm (Auto Qualified Lead)

Idempotency Rules (Chống điểm phồng)

Bắt buộc implement để tránh lead score bị thổi phồng do submit nhiều lần:

  • Assessment: Một email chỉ được cộng +20 điểm một lần duy nhất (lần submit đầu tiên). Các lần submit tiếp theo vẫn tạo bản ghi assessments mới (để so sánh tiến độ) nhưng không cộng thêm điểm. Backend kiểm tra: SELECT COUNT(*) FROM assessments WHERE lead_id = ? , nếu > 0 thì bỏ qua bước cộng điểm.
  • Gated Content: Một email chỉ được cộng +15 điểm cho mỗi resource_slug là duy nhất. Tải lại cùng file PDF nhiều lần không cộng thêm. Backend kiểm tra: SELECT COUNT(*) FROM lead_events WHERE lead_id = ? AND event_type = 'download_resource' AND resource_slug = ?, nếu > 0 thì skip.
  • Strategy Call: Một email chỉ được đặt trạng thái 'SQL' một lần. Nếu đã là 'SQL', không cộng thêm 50 điểm nữa.

6. Information Architecture & AEO/SEO Specs

Để tối ưu cho Google AI Overviews / AI Mode, ChatGPT Search, và Perplexity, website tuyệt đối KHÔNG thiết kế kiểu "1 Landing Page cuộn dọc". Sunext phải là một Entity Graph chuyên gia với các URL rõ ràng, Schema chuẩn và nội dung public-crawlable. Theo Google Search Central, không có yêu cầu kỹ thuật riêng để xuất hiện trong AI Overviews — website chỉ cần index được, có snippet hữu ích, và được hiểu rõ bởi Search.

-- 1. Bảng companies
CREATE TABLE companies (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  domain VARCHAR(255) UNIQUE NOT NULL,

  size_range VARCHAR(50),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_companies_domain ON companies(domain);

-- 2. Bảng leads (Khách hàng)
CREATE TABLE leads (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  email VARCHAR(255) UNIQUE NOT NULL, 
  full_name VARCHAR(255),
  company_id UUID REFERENCES companies(id) ON DELETE SET NULL,
  total_score INT DEFAULT 0, 
  status VARCHAR(50) DEFAULT 'New', 
  marketing_consent BOOLEAN DEFAULT false,
  consent_timestamp TIMESTAMPTZ,
  consent_source VARCHAR(255), -- url nơi user tick consent
  utm_source VARCHAR(100),
  utm_campaign VARCHAR(100),
  last_activity_at TIMESTAMPTZ DEFAULT NOW(),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  deleted_at TIMESTAMPTZ -- cho soft delete
);
CREATE INDEX idx_leads_email ON leads(email);
CREATE INDEX idx_leads_company_id ON leads(company_id);

-- 3. Bảng assessments
CREATE TABLE assessments (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  lead_id UUID REFERENCES leads(id) ON DELETE CASCADE,
  answers_json JSONB NOT NULL,
  calculated_score INT NOT NULL,
  recommended_tier INT,
  recommended_programs JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_assessments_lead_id ON assessments(lead_id);

-- 4. Bảng form_submissions
CREATE TABLE form_submissions (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  lead_id UUID REFERENCES leads(id) ON DELETE CASCADE,
  form_type VARCHAR(50) NOT NULL, 
  payload JSONB NOT NULL,
  ip_address VARCHAR(50), 
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 5. Bảng lead_events (Phân biệt với score events)
CREATE TABLE lead_events (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  lead_id UUID REFERENCES leads(id) ON DELETE CASCADE,
  event_type VARCHAR(100) NOT NULL, -- VD: 'download_resource', 'page_view'
  resource_slug VARCHAR(255),
  score_added INT DEFAULT 0, 
  created_at TIMESTAMPTZ DEFAULT NOW()
);
-- Constraint Idempotency cho Gated Content (1 email tải 1 file chỉ có 1 event download)
CREATE UNIQUE INDEX idx_lead_events_idempotency 
ON lead_events(lead_id, event_type, resource_slug) 
WHERE event_type = 'download_resource';

-- 6. Bảng webhook_logs
CREATE TABLE webhook_logs (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  target_system VARCHAR(50),

  response_status INT,
  error_message TEXT,
  retry_count INT DEFAULT 0,
  webhook_status VARCHAR(20) DEFAULT 'pending', -- 'success', 'failed'
  created_at TIMESTAMPTZ DEFAULT NOW()

9. API Contracts & Webhooks (BFF)

Mọi API route (Next.js) đều yêu cầu Turnstile token (cf-turnstile-response) và Rate Limit dựa trên IP (qua Vercel KV hoặc middleware).

1. POST /api/assessment/submit

Rate Limit: 5 req / 10 min / IP. Auth: Turnstile.

Request Payload:

{
  "email": "a@fpt.com", "fullName": "Nguyen A",
  "answers": { "q1": "c", "q2": "b", "q3": "c", "q4": "a", "q5": "b", "q6": "a", "q7": "c", "q8": "b", "q9": "c", "q10": "b" },
  "utm": { "source": "fb" },
  "cf-turnstile-response": "..."
}

Logic Backend & Idempotency: 1. Validate Turnstile.
2. Upsert companies (fpt.com).
3. Check DB assessments: Nếu chưa có assessment nào của lead_id này -> Upsert leads (cộng +20đ). Nếu đã có -> không cộng điểm.
4. Tính điểm, insert vào assessments.
5. Gọi SendGrid API gửi email. Lưu webhook_logs.
6. Trả về kết quả 200 OK.

Response (200 OK): { "success": true, "data": { "score": 65, "recommended_tier": 3, "recommended_programs": ["agentics-ai-automations", "ai-agent-bootcamp"] } }

Error Responses (Tương tự cho cả 3 API):

// 400: Lỗi input (Email cá nhân, thiếu field)
{ "success": false, "error": "VALIDATION_ERROR", "message": "Email công ty là bắt buộc." }
// 401: Turnstile fail
{ "success": false, "error": "CAPTCHA_FAILED", "message": "Xác minh thất bại. Vui lòng thử lại." }
// 429: Rate limit
{ "success": false, "error": "RATE_LIMITED", "message": "Thử lại sau 5 phút.", "retryAfterSeconds": 300 }
// 500: Lỗi Server/DB (không trả chi tiết DB ra client)
{ "success": false, "error": "INTERNAL_ERROR", "message": "Có lỗi hệ thống." }

10. Cấu Hình Bảo Mật & Non-Functional Requirements

10.1. Supabase Row Level Security (RLS) Policy

Bắt buộc thiết lập RLS trên tất cả bảng chứa dữ liệu Lead trước khi Go-live. Nguyên tắc: anon role không bao giờ được phép SELECT dữ liệu Lead.

-- Bật RLS trên tất cả bảng nhạy cảm
ALTER TABLE leads ENABLE ROW LEVEL SECURITY;
ALTER TABLE assessments ENABLE ROW LEVEL SECURITY;
ALTER TABLE form_submissions ENABLE ROW LEVEL SECURITY;
ALTER TABLE lead_events ENABLE ROW LEVEL SECURITY;
ALTER TABLE webhook_logs ENABLE ROW LEVEL SECURITY;

-- KHÔNG tạo policy SELECT cho anon. Mặc định: deny all.
-- Chỉ cho phép service_role (dùng trong BFF/API Routes) toàn quyền.
-- Frontend Client-side TUYỆT ĐỐI không được gọi Supabase trực tiếp vào các bảng này.
-- Mọi INSERT/UPDATE phải đi qua Next.js API Routes (BFF) dùng SUPABASE_SERVICE_ROLE_KEY.

Biến môi trường bắt buộc phân tách

  • NEXT_PUBLIC_SUPABASE_URL — Dùng ở client-side (an toàn).
  • NEXT_PUBLIC_SUPABASE_ANON_KEY — Chỉ dùng cho các tác vụ public (đọc trang tĩnh, không liên quan Lead).
  • SUPABASE_SERVICE_ROLE_KEY — TUYỆT ĐỐI không đặt vào NEXT_PUBLIC_*. Chỉ dùng trong process.env của API Routes server-side.
